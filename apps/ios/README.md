# Agent Room iOS

`AgentRoomMobile` is a SwiftPM package for the iPhone client surface. It is
structured so the model/API/UI code can be tested from the command line and then
embedded into an Xcode iOS App target.

## What It Supports

- Configure a relay origin. The sample app defaults to `http://127.0.0.1:8080/`.
- Join an existing room by ID.
- Create a room through `POST /v1/rooms`.
- Load recent messages through `GET /v1/rooms/{room}/messages`.
- Load participants through `GET /v1/rooms/{room}/participants`.
- Send chat messages through `POST /v1/rooms/{room}/messages`.
- Subscribe to room updates through `/v1/rooms/{room}/ws`.

## Local Checks

From this directory:

```bash
cd apps/ios/AgentRoomMobile
swift test
```

## Xcode App Target

`AgentRoomMobileApp` is the local simulator app target generated by XcodeGen.

```bash
cd apps/ios/AgentRoomMobileApp
xcodegen generate
xcodebuild -project AgentRoomMobileApp.xcodeproj \
  -target AgentRoomMobileApp \
  -sdk iphonesimulator26.5 \
  -configuration Debug build
```

The current local smoke path uses an already-installed iOS simulator runtime:

```bash
xcrun simctl boot E8B04BC8-5F8A-41AC-B6A5-D32226C82322
xcrun simctl install E8B04BC8-5F8A-41AC-B6A5-D32226C82322 \
  apps/ios/AgentRoomMobileApp/build/Debug-iphonesimulator/AgentRoomMobileApp.app
xcrun simctl launch E8B04BC8-5F8A-41AC-B6A5-D32226C82322 com.agentroom.mobile
```

To create a fresh Xcode target manually instead:

1. Open Xcode and create an iOS App target named `AgentRoomMobileApp`.
2. Add this folder as a local Swift Package dependency.
3. Set the app's root scene to `AgentRoomRootView()`.
4. Add `NSAppTransportSecurity` exceptions if you need to connect to a local
   non-HTTPS relay from a physical phone.

For simulator testing against a relay running on the Mac, use:

```text
http://127.0.0.1:8080
```

For a physical iPhone on the same LAN, use the Mac's LAN IP instead:

```text
http://<mac-lan-ip>:8080
```
